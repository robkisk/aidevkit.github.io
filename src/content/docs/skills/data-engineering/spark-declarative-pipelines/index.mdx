## Overview

The Spark Declarative Pipelines (SDP) skill teaches Claude Code how to create, configure, and manage Databricks Lakeflow pipelines using serverless compute. Use it when building medallion architectures, ingesting streaming data, implementing change data capture (CDC), or working with streaming tables and materialized views. It supports both SQL and Python pipeline definitions.

## Capabilities

- Create new pipeline projects using `databricks pipelines init` with Databricks Asset Bundles
- Build streaming tables and materialized views in SQL or Python
- Ingest data from cloud storage with Auto Loader (`read_files` / `cloudFiles`)
- Implement Change Data Capture (CDC) with AUTO CDC for SCD Type 1 and Type 2
- Design medallion architectures (bronze/silver/gold) with proper layering
- Configure Liquid Clustering for optimized query performance
- Set up multi-environment deployments (dev/staging/prod)
- Handle schema evolution, expectations (data quality rules), and error handling
- Write to external systems with Sinks and ForEachBatch Sinks (Python only)
- Migrate legacy DLT pipelines to the modern SDP API

## Key Concepts

### Streaming Tables vs Materialized Views

**Streaming tables** process data incrementally -- each run picks up only new records from the source. Best for append-only ingestion from files, Kafka, or Kinesis.

**Materialized views** recompute their full result set on each refresh. Best for aggregations, joins, and any transformation where the output depends on the entire input dataset.

### Auto Loader

Auto Loader (`read_files` in SQL, `cloudFiles` in Python) incrementally ingests new files from cloud storage. It tracks which files have been processed and only reads new arrivals, making it ideal for landing zones and raw data volumes.

### Medallion Architecture

A layered data design pattern:
- **Bronze** -- Raw ingestion, minimal transformation, append-only
- **Silver** -- Cleaned and validated, business logic applied, deduplication
- **Gold** -- Aggregated and denormalized, optimized for BI and reporting

### AUTO CDC and SCD Type 2

AUTO CDC applies change data capture operations (inserts, updates, deletes) from a source to a target table. When configured for SCD Type 2, it maintains a full history of changes with `__START_AT` and `__END_AT` timestamp columns.

### Liquid Clustering

Replaces traditional `PARTITION BY` with dynamic `CLUSTER BY` that automatically optimizes data layout. Use it on columns frequently used in filters and joins.

### Decision Tree

```
User request -> What kind of output?
├── Intermediate logic (not persisted) -> Temporary View
├── Persisted dataset
│   ├── Source is streaming/incremental -> Streaming Table
│   │   ├── File ingestion -> Auto Loader
│   │   ├── CDC / upserts / SCD -> Auto CDC
│   │   └── Multiple sources into one table -> Append Flows
│   └── Source is batch/full scan -> Materialized View
│       ├── Aggregation/join across full dataset
│       └── Gold layer from streaming table (batch read)
├── Output to external system (Python only) -> Sink
└── Data quality constraints -> Expectations (any dataset)
```

## Pipeline Structure

```
my-pipeline-project/
├── databricks.yml
├── resources/
│   ├── my_pipeline.pipeline.yml
│   └── my_pipeline_job.job.yml
└── src/
    ├── bronze.py (or .sql)
    ├── silver.py (or .sql)
    └── gold.py (or .sql)
```
