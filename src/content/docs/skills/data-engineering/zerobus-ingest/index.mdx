## Overview

The Zerobus Ingest skill teaches Claude Code how to build producers that write records directly into Unity Catalog Delta tables via gRPC, eliminating the need for message bus infrastructure (Kafka, Kinesis, Event Hub). It covers SDK setup across five languages, Protobuf schema generation from table definitions, ACK handling, retry logic, and operational limits.

**Status:** GA (Generally Available since February 2026; billed under Lakeflow Jobs Serverless SKU)

## Capabilities

- Configure Zerobus server endpoints for AWS and Azure regions
- Create and grant service principals for M2M authentication
- Build Python producers with sync and async APIs
- Build producers in Java, Go, TypeScript, and Rust
- Generate Protobuf schemas from Unity Catalog table definitions
- Implement ACK callbacks for high-throughput ingestion
- Handle retries with exponential backoff and stream reinitialization
- Monitor producer health and ingestion lag

## Key Concepts

### Core Pattern

SDK init -> create stream -> ingest records -> handle ACKs -> flush -> close

### JSON vs Protobuf

| Aspect | JSON | Protobuf |
|--------|------|----------|
| Type safety | None | Compile-time checking |
| Schema evolution | Manual | Forward-compatible |
| Performance | Text parsing overhead | Binary, smaller payloads |
| Recommended for | Prototyping | Production |

### Delivery Semantics

Zerobus provides **at-least-once** delivery. Design consumers to handle duplicate records.

### Serverless Limitation

The Zerobus SDK cannot pip-install on serverless compute. Use classic compute clusters, or use the Zerobus REST API (Beta) for notebook-based ingestion.

## Minimal Python Example

```python
import json
from zerobus.sdk.sync import ZerobusSdk
from zerobus.sdk.shared import RecordType, StreamConfigurationOptions, TableProperties

sdk = ZerobusSdk(server_endpoint, workspace_url)
options = StreamConfigurationOptions(record_type=RecordType.JSON)
table_props = TableProperties(table_name)

stream = sdk.create_stream(client_id, client_secret, table_props, options)
try:
    record = {"device_name": "sensor-1", "temp": 22, "humidity": 55}
    stream.ingest_record(json.dumps(record))
    stream.flush()
finally:
    stream.close()
```

## Common Issues

| Issue | Solution |
|-------|----------|
| Connection refused | Verify endpoint format matches your cloud (AWS vs Azure) |
| Authentication failed | Confirm service principal grants (`MODIFY` + `SELECT` on table) |
| Schema mismatch | Regenerate .proto if table changed |
| SDK install fails on serverless | Use classic compute or REST API |
| Error 4024 / authorization_details | Grant explicit table-level permissions |
