## Server Endpoint

| Cloud | Format |

| **Azure** | `<workspace-id>.zerobus.<region>.azuredatabricks.net` |

## Create the Target Table

Zerobus does not create or alter tables. Pre-create a managed Delta table:

```sql
CREATE TABLE catalog.schema.my_events (
    event_id STRING, device_name STRING,
    temp INT, humidity LONG, event_time TIMESTAMP
);
```

Constraints: managed Delta only, ASCII names, max 2000 columns, supported region.

## Service Principal Setup

```sql
GRANT USE CATALOG ON CATALOG my_catalog TO `<sp-uuid>`;
GRANT USE SCHEMA ON SCHEMA my_catalog.my_schema TO `<sp-uuid>`;
GRANT MODIFY, SELECT ON TABLE my_catalog.my_schema.my_events TO `<sp-uuid>`;
```

Always grant explicit table-level `MODIFY` and `SELECT` -- schema-level inherited grants may not suffice for the OAuth `authorization_details` flow.

## Environment Variables

```bash
export ZEROBUS_SERVER_ENDPOINT="<workspace-id>.zerobus.<region>.cloud.databricks.com"
export DATABRICKS_WORKSPACE_URL="https://<instance>.cloud.databricks.com"
export ZEROBUS_TABLE_NAME="my_catalog.my_schema.my_events"
export DATABRICKS_CLIENT_ID="<client-id>"
export DATABRICKS_CLIENT_SECRET="<client-secret>"
```
