## Overview

The Spark Structured Streaming skill teaches Claude Code how to build production-ready streaming pipelines on Databricks. Use it when building Kafka ingestion pipelines, implementing Real-Time Mode for sub-second latency, configuring triggers and checkpoints, performing stream-stream or stream-static joins, writing to multiple sinks, or tuning streaming cost and performance.

## Capabilities

- Build Kafka-to-Delta ingestion pipelines with schema validation
- Configure Real-Time Mode (RTM) for sub-second latency (as low as 5ms)
- Set up `processingTime`, `availableNow`, and `realTime` triggers
- Implement stream-stream joins with watermarks and state management
- Enrich streaming data with Delta dimension tables (stream-static joins)
- Write a single stream to multiple Delta tables or Kafka topics
- Perform parallel MERGE operations with Liquid Clustering
- Configure checkpoints for exactly-once semantics
- Manage stateful operations with watermarks and RocksDB state stores
- Optimize streaming costs through trigger tuning and cluster right-sizing

## Key Concepts

### Trigger Types

| Trigger | Latency | Cost | Use Case |
|---------|---------|------|----------|
| `realTime` | < 800ms | $$$ | Sub-second requirements |
| `processingTime("30 seconds")` | Seconds | $$ | Near real-time dashboards |
| `availableNow=True` (scheduled) | Minutes | $ | Batch-style SLA |

### Stream-Static vs Stream-Stream Joins

**Stream-static joins** are stateless -- the dimension table refreshes each microbatch automatically when stored as Delta. Use `left` joins to prevent data loss.

**Stream-stream joins** are stateful -- both sides buffer events until matches are found or watermarks expire. Always include explicit time bounds in the join condition.

### Checkpoints

Every streaming query needs a unique, persistent checkpoint location. Use Unity Catalog volumes (not DBFS). Tie checkpoints to the target table, not the source.

### Watermarks

Watermarks control state cleanup for stateful operations (deduplication, aggregations, stream-stream joins). Rule of thumb: start with 2-3x your p95 event latency.

## Production Checklist

- Checkpoint location is persistent (UC volumes, not DBFS)
- Unique checkpoint per stream
- Fixed-size cluster (no autoscaling for streaming)
- Monitoring configured (input rate, lag, batch duration)
- Exactly-once verified (txnVersion/txnAppId)
- Watermark configured for stateful operations
- Left joins for stream-static (not inner)
