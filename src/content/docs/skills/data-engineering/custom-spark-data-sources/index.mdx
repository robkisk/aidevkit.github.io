## Overview

The Custom Spark Data Sources skill teaches Claude Code how to build Python connectors for external systems using the PySpark DataSource API. Use it when Spark lacks a native connector for your target system (database, API, message queue, custom protocol) and you need batch or streaming read/write capabilities.

## Capabilities

- Implement batch readers and writers using the `DataSource` API
- Implement streaming readers and writers with offset tracking
- Design partitioning strategies (time-based, token-range, ID-range)
- Handle authentication with priority-based credential chains
- Map types bidirectionally between Spark and external systems
- Add retry logic, circuit breakers, and dead letter queues
- Write unit and integration tests with mocked external systems
- Harden connectors for production with observability and security

## Key Concepts

### Architecture

Each data source follows a flat, single-level inheritance structure:

1. **DataSource class** -- entry point that returns readers/writers
2. **Base Reader/Writer** -- shared logic for options and data processing
3. **Batch classes** -- inherit from base + `DataSourceReader`/`DataSourceWriter`
4. **Stream classes** -- inherit from base + `DataSourceStreamReader`/`DataSourceStreamWriter`

### Design Constraints

- **Flat single-level inheritance** -- PySpark serializes reader/writer instances to executors; complex hierarchies break serialization
- **Import inside executor methods** -- `read()` and `write()` run on remote processes; top-level imports are not available
- **Minimize dependencies** -- every package must be installed on all executor nodes
- **No async/await** unless the external system's SDK is async-only
