**Skill:** `databricks-unity-catalog`

## Overview

The Unity Catalog skill teaches Claude Code how to work with Unity Catalog system tables (audit, lineage, billing, compute, jobs), volume file operations (upload, download, list), and Data Profiling (quality monitoring with drift detection). Use it when querying operational metadata, managing files in UC Volumes, or setting up data quality monitors.

## Capabilities

- Query system tables for audit logs, lineage tracking, billing analysis, and job monitoring
- Upload, download, and list files in Unity Catalog Volumes
- Create data quality monitors with profile, drift, and anomaly detection
- Monitor ML model performance with inference tables

## Key Concepts

### Three-Level Namespace

All Unity Catalog objects follow `catalog.schema.object` naming:

- **Catalog** -- top-level container
- **Schema** -- logical grouping within a catalog
- **Table/View/Volume/Function** -- data objects within a schema

### System Tables

System tables are read-only tables under the `system` catalog that provide operational metadata. They require account admin or metastore admin privileges to enable.

### Volumes

Volumes provide governed file storage in Unity Catalog. Two types:
- **Managed** -- storage location managed by UC
- **External** -- user-provided cloud storage

### Data Profiling

Create quality monitors on UC tables to track data profiles, detect drift, and monitor ML model performance. The Python SDK package is `databricks.lakehouse_monitoring`.
