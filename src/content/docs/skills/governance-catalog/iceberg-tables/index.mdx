## Overview

The Iceberg Tables skill teaches Claude Code how to work with Apache Iceberg on Databricks. It covers creating native managed Iceberg tables, enabling UniForm (External Iceberg Reads) on Delta tables, using Compatibility Mode for streaming tables and materialized views, configuring the Iceberg REST Catalog (IRC) for external engine access, Snowflake interoperability, and connecting PyIceberg, OSS Spark, EMR, and Flink.

## Capabilities

- Create managed Iceberg tables with `USING ICEBERG` DDL
- Enable UniForm on existing Delta tables for external Iceberg reads
- Configure Compatibility Mode for streaming tables and materialized views in SDP pipelines
- Set up the Iceberg REST Catalog endpoint for external engines
- Integrate bidirectionally with Snowflake via catalog integration
- Connect PyIceberg, OSS Spark, AWS EMR, Flink, and Kafka Connect

## Key Concepts

| Concept | Summary |
|---------|---------|
| **Managed Iceberg Table** | Native Iceberg table (`USING ICEBERG`) -- full read/write in Databricks and external engines |
| **External Iceberg Reads (UniForm)** | Delta table with auto-generated Iceberg metadata -- read as Iceberg externally |
| **Compatibility Mode** | UniForm for streaming tables and MVs in SDP pipelines |
| **Iceberg REST Catalog (IRC)** | Unity Catalog's REST endpoint for external engine access |
| **Iceberg v3** | Next-gen format (Beta, DBR 17.3+) with deletion vectors and VARIANT type |

## Quick Start

```sql
-- Managed Iceberg table with partitioning
CREATE TABLE my_catalog.my_schema.events
USING ICEBERG
PARTITIONED BY (event_date)
AS SELECT * FROM raw_events;

-- Enable UniForm on existing Delta table
ALTER TABLE my_catalog.my_schema.customers
SET TBLPROPERTIES (
  'delta.columnMapping.mode' = 'name',
  'delta.enableIcebergCompatV2' = 'true',
  'delta.universalFormat.enabledFormats' = 'iceberg'
);
```

## Read/Write Matrix

| Table Type | Databricks R/W | External IRC Read | External IRC Write |
|------------|:-:|:-:|:-:|
| Managed Iceberg | Yes/Yes | Yes | Yes |
| Delta + UniForm | Yes/Yes | Yes | No |
| Delta + Compatibility Mode | Yes/Yes | Yes | No |

## Critical Rules

- All Iceberg features require Unity Catalog-enabled workspaces
- Do not install an Iceberg library into DBR (built-in support)
- Do not set `write.metadata.path` or `write.metadata.previous-versions-max`
- Expression transforms (`bucket()`, `years()`) are not supported with `PARTITIONED BY`
- Disable deletion vectors when using `CLUSTER BY` on v2: `'delta.enableDeletionVectors' = false`

## Common Issues

| Issue | Solution |
|-------|----------|
| No Change Data Feed (CDF) | Not supported on managed Iceberg; use Delta + UniForm |
| UniForm async delay | Iceberg metadata is async; brief delay before external reads |
| Compression codec | Managed Iceberg uses `zstd` by default; verify reader compatibility |
| No shallow clone | Use `DEEP CLONE` or `CREATE TABLE AS SELECT` |
