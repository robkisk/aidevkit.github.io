## Chart Data Modes

All chart/data components support two modes:

- **Query mode:** pass `queryKey` + `parameters` -- component fetches data automatically
- **Data mode:** pass static data via `data` prop -- no `queryKey`/`parameters` needed

```tsx
// Query mode (recommended for Databricks SQL)
<BarChart queryKey="sales_by_region" parameters={{}} />

// Data mode (static/pre-fetched data)
<BarChart data={myData} xKey="category" yKey="count" />
```

`parameters` is **required** even when empty (`parameters={{}}`).

## Common Anti-Patterns

| Mistake | Why | Fix |
|---------|-----|-----|
| `xAxisKey`, `dataKey` | Recharts naming | Use `xKey`, `yKey` |
| `<XAxis>`, `<YAxis>` children | ECharts-based, not Recharts | Configure via props only |
| `columns` on DataTable | Auto-generates columns | Use `queryKey` + `parameters` |
| Double-fetching | Components handle their own fetching | Just pass `queryKey` |

## Data Components

| Component | Extra Props | Use For |
|-----------|-------------|---------|
| `BarChart` | `xKey`, `yKey`, `orientation`, `stacked` | Categorical comparisons |
| `LineChart` | `xKey`, `yKey`, `smooth`, `showSymbol` | Time series, trends |
| `AreaChart` | `xKey`, `yKey`, `smooth`, `stacked` | Cumulative trends |
| `PieChart` | `xKey`, `yKey`, `innerRadius`, `showLabels` | Part-of-whole |
| `ScatterChart` | `xKey`, `yKey`, `symbolSize` | Correlation |
| `HeatmapChart` | `xKey`, `yKey`, `min`, `max` | Matrix-style |
| `DataTable` | `filterColumn`, `pageSize`, `transform` | Tabular data |

## UI Components

| Component | Common Props |
|-----------|-------------|
| `Card`, `CardHeader`, `CardTitle`, `CardContent` | Standard container |
| `Badge` | `variant`: default, secondary, destructive, outline |
| `Button` | `variant`, `size`, `onClick` |
| `Select`, `SelectTrigger`, `SelectContent`, `SelectItem` | Dropdown (value cannot be "") |
| `Tabs`, `TabsList`, `TabsTrigger`, `TabsContent` | Tabbed interface |
| `Skeleton` | Loading states |

## useAnalyticsQuery Hook

Use only for custom layouts (KPIs, cards). For charts/tables, pass `queryKey` directly.

```typescript
// Memoize parameters to prevent infinite loops
const params = useMemo(() => ({ id: sql.string(selectedId) }), [selectedId]);
const { data, loading, error } = useAnalyticsQuery('query', params);

// Conditional queries: use autoStart: false
const { data } = useAnalyticsQuery('query', params, { autoStart: false });
```
