## Overview

The Databricks Apps Python skill teaches Claude Code how to build, configure, and deploy Python web applications on the Databricks Apps platform. It covers six pre-installed frameworks (Dash, Streamlit, Gradio, Flask, FastAPI, Reflex), service principal and user authorization models, resource binding via `app.yaml`, Lakebase PostgreSQL connectivity, foundation model API integration, and deployment via CLI or Asset Bundles.

## Capabilities

- Select the right framework for dashboards, prototypes, ML demos, APIs, or full-stack apps
- Configure service principal (app) auth and on-behalf-of (user) auth with OAuth scopes
- Bind SQL warehouses, Lakebase databases, model serving endpoints, secrets, and volumes via `valueFrom`
- Connect to Lakebase (PostgreSQL) with psycopg2, asyncpg, or SQLAlchemy
- Deploy apps using Databricks CLI or Asset Bundles (DABs) with multi-environment targets
- Integrate Databricks foundation model APIs using the OpenAI-compatible client
- Manage app lifecycle programmatically via MCP tools

## Framework Selection

| Framework | Best For | app.yaml Command |
|-----------|----------|------------------|
| **Dash** | Production dashboards, BI tools, complex interactivity | `["python", "app.py"]` |
| **Streamlit** | Rapid prototyping, data science apps, internal tools | `["streamlit", "run", "app.py"]` |
| **Gradio** | ML demos, model interfaces, chat UIs | `["python", "app.py"]` |
| **Flask** | Custom REST APIs, lightweight apps, webhooks | `["gunicorn", "app:app", "-w", "4", "-b", "0.0.0.0:8000"]` |
| **FastAPI** | Async APIs, auto-generated OpenAPI docs | `["uvicorn", "app:app", "--host", "0.0.0.0", "--port", "8000"]` |
| **Reflex** | Full-stack Python apps without JavaScript | `["reflex", "run", "--env", "prod"]` |

**Default recommendation:** Streamlit for prototypes, Dash for production dashboards, FastAPI for APIs, Gradio for ML demos.

## Platform Constraints

| Constraint | Details |
|------------|---------|
| **Runtime** | Python 3.11, Ubuntu 22.04 LTS |
| **Compute** | 2 vCPUs, 6 GB memory (default) |
| **Pre-installed** | Dash 2.18.1, Streamlit 1.38.0, Gradio 4.44.0, Flask 3.0.3, FastAPI 0.115.0 |
| **Custom packages** | Add to `requirements.txt` in app root |
| **Port** | Apps must bind to `DATABRICKS_APP_PORT` env var (defaults to 8000) |

### SQL Warehouse Connection

```python
from databricks.sdk.core import Config
from databricks import sql

cfg = Config()  # Auto-detects credentials from environment
conn = sql.connect(
    server_hostname=cfg.host,
    http_path=f"/sql/1.0/warehouses/{os.getenv('DATABRICKS_WAREHOUSE_ID')}",
    credentials_provider=lambda: cfg.authenticate,
)
```

## Common Issues

| Issue | Solution |
|-------|----------|
| **Connection exhausted** | Use `@st.cache_resource` (Streamlit) or connection pooling |
| **Auth token not found** | Check `x-forwarded-access-token` header -- only available when deployed |
| **App won't start** | Check `app.yaml` command matches framework |
| **Lakebase crash on start** | `psycopg2`/`asyncpg` are NOT pre-installed -- add to `requirements.txt` |
| **Port conflict** | Bind to `DATABRICKS_APP_PORT` env var (never use 8080) |
