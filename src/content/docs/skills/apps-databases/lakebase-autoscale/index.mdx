**Skill:** `databricks-lakebase-autoscale`

## Overview

The Lakebase Autoscaling skill teaches Claude Code how to create and manage Lakebase Autoscaling projects -- Databricks' next-generation managed PostgreSQL service for OLTP workloads. It covers project hierarchy (projects, branches, computes, databases), autoscaling compute from 0.5 to 112 CU, Git-like branching for dev/test workflows, scale-to-zero cost savings, OAuth authentication with token refresh, reverse ETL via synced tables, and CLI management.

## Capabilities

- Create and manage Lakebase Autoscaling projects with Python SDK or CLI
- Configure autoscaling compute (0.5-112 CU) with scale-to-zero
- Create isolated database branches for dev/test/staging workflows
- Connect applications with OAuth tokens and automatic token refresh
- Sync data from Delta Lake to PostgreSQL via reverse ETL (synced tables)
- Manage projects, branches, endpoints, roles, and databases via `databricks postgres` CLI

## Key Concepts

### Project Hierarchy

```
Project (top-level container)
  |-- Branch(es) (isolated database environments)
        |-- Compute (primary R/W endpoint)
        |-- Read Replica(s) (optional, read-only)
        |-- Database(s) (Postgres databases)
```

### Quick Start

```python
from databricks.sdk import WorkspaceClient
from databricks.sdk.service.postgres import Project, ProjectSpec

w = WorkspaceClient()
operation = w.postgres.create_project(
    project=Project(spec=ProjectSpec(display_name="My App", pg_version="17")),
    project_id="my-app"
)
result = operation.wait()
```

### CLI Quick Reference

```bash
databricks postgres create-project --project-id my-app \
    --json '{"spec": {"display_name": "My App", "pg_version": "17"}}'
databricks postgres list-projects
databricks postgres list-branches projects/my-app
databricks postgres get-endpoint projects/my-app/branches/production/endpoints/ep-primary
```

### Autoscaling vs Provisioned

| Aspect | Provisioned | Autoscaling |
|--------|-------------|-------------|
| SDK module | `w.database` | `w.postgres` |
| Capacity | CU_1 to CU_8 (16 GB/CU) | 0.5-112 CU (2 GB/CU) |
| Branching | Not supported | Full support |
| Scale-to-zero | Not supported | Configurable timeout |
| Operations | Synchronous | Long-running (`.wait()`) |

## Common Issues

| Issue | Solution |
|-------|----------|
| Token expired | Implement token refresh; tokens expire after 1 hour |
| Connection refused after scale-to-zero | Compute wakes automatically; implement retry logic |
| DNS resolution fails on macOS | Use `dig` to resolve, pass `hostaddr` to psycopg |
| Branch deletion blocked | Delete child branches first |
