**Skill:** `databricks-jobs`

## Overview

The Jobs Orchestration skill teaches Claude Code how to create and manage Databricks Lakeflow Jobs via CLI, Python SDK, and Asset Bundles. It covers task types (notebook, Spark Python, SQL, SDP pipeline, dbt), triggers and scheduling (cron, periodic, file arrival, table/storage change), notifications (email, webhook, PagerDuty, Slack), health rules, budgets, and multi-task pipelines with dependencies.

This skill also covers scaffolding new job projects with `databricks bundle init` for CI/CD-ready deployments.

## Capabilities

- Create jobs with notebook, Python, SQL, SDP pipeline, and dbt tasks
- Configure cron schedules, periodic triggers, and event-based triggers
- Set up email, webhook, PagerDuty, and custom notifications
- Define multi-task pipelines with sequential and parallel dependencies
- Use health rules for SLA monitoring and automatic failure detection
- Scaffold job projects with `databricks bundle init` for DABs integration
- Manage jobs via CLI, Python SDK, or MCP tools

## Key Concepts

### Job Scaffolding (DABs)

```bash
databricks bundle init default-python --config-file <(echo '{
  "project_name": "my_job",
  "include_job": "yes",
  "include_pipeline": "no",
  "include_python": "yes",
  "serverless": "yes"
}') --profile <PROFILE> < /dev/null
```

### Task Types

| Task Type | Use Case |
|-----------|----------|
| Notebook | Interactive notebooks with parameters |
| Spark Python | Python scripts on Spark |
| SQL | SQL statements against warehouses |
| SDP Pipeline | SDP pipeline orchestration |
| dbt | dbt project execution |
| Python Wheel | Packaged Python applications |

### Trigger Types

| Trigger | Use Case |
|---------|----------|
| Cron schedule | Fixed recurring schedules |
| Periodic | Run every N time units |
| File arrival | Trigger on new files in cloud storage |
| Table update | Trigger on Delta table changes |

### Compute Options

| Compute | When to Use |
|---------|-------------|
| Serverless | Default choice, no cluster management |
| Job cluster | Custom configurations, GPU, specific libraries |
| Existing cluster | Interactive development, shared compute |
