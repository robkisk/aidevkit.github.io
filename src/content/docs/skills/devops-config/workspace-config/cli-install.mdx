The Databricks CLI is a prerequisite for bundle deployments, OAuth authentication, and workspace administration from the terminal. This page covers installation on macOS, Windows, and Linux along with version verification and common troubleshooting steps.

## Installation Methods

### macOS (Homebrew)

```bash
brew tap databricks/tap
brew install databricks
```

### Linux (curl)

```bash
curl -fsSL https://raw.githubusercontent.com/databricks/setup-cli/main/install.sh | sh
```

### Windows (WinGet)

```powershell
winget install Databricks.DatabricksCLI
```

## Verification

```bash
databricks --version
# Expected: Databricks CLI v0.292.0 or later
```

## Sandboxed Environments

For IDE containers or sandboxed environments without Homebrew:

```bash
# User directory install (no sudo)
curl -fsSL https://raw.githubusercontent.com/databricks/setup-cli/main/install.sh | sh -s -- --install-dir ~/.local/bin
```

## Minimum Version

Many features require Databricks CLI >= v0.292.0. Update with:

```bash
brew upgrade databricks  # macOS
# or re-run the install script for Linux
```
