## Overview

The Python SDK skill teaches Claude Code how to use the `databricks-sdk` package to interact with Databricks workspace and account APIs programmatically. Use it when automating infrastructure management, running SQL queries, managing Unity Catalog objects, querying model serving endpoints, or building applications that integrate with Databricks. It covers authentication patterns, all major API surfaces, and async usage for web frameworks.

## Capabilities

- Authenticate using environment variables, profiles, tokens, or Azure service principals
- Manage clusters (create, start, stop, resize, autoscale)
- Create and run jobs with notebook, Python, and SQL tasks
- Execute SQL statements against warehouses with parameterized queries
- Manage Unity Catalog objects (catalogs, schemas, tables, volumes)
- Upload and download files to/from Unity Catalog Volumes
- Query and manage model serving endpoints (LLMs, custom models, embeddings)
- Work with vector search endpoints and indexes
- Manage SDP pipelines (create, start, stop, monitor)
- Access secrets and DBUtils from local development
- Make direct REST API calls for unsupported endpoints
- Use the SDK safely in async applications (FastAPI, asyncio)

## Key Concepts

### Authentication

The SDK supports multiple authentication methods. Environment variables are recommended for most cases:

| Method | When to Use |
|--------|-------------|
| Environment variables (`DATABRICKS_HOST`, `DATABRICKS_TOKEN`) | Default for local development and CI/CD |
| Named profile (`~/.databrickscfg`) | Multiple workspaces on one machine |
| Explicit token | Programmatic configuration |
| Azure service principal | Azure deployments and automation |
| Notebook context | Auto-detected inside Databricks notebooks |

### Async Safety

The Databricks SDK is fully synchronous. In async applications (FastAPI, asyncio), wrap all SDK calls with `asyncio.to_thread()` to avoid blocking the event loop. Property access like `w.config.host` is safe without wrapping.

### Long-Running Operations

Many SDK methods return `Wait` objects for operations that take time (cluster creation, job runs). Use the `*_and_wait()` convenience methods for blocking calls, or call `.result()` on the `Wait` object to block until completion.

### Pagination

All `list` methods return iterators that handle pagination automatically. Simply iterate with a `for` loop to fetch all pages transparently.

### Error Handling

Import specific exception types from `databricks.sdk.errors`:
- `NotFound` — Resource does not exist
- `PermissionDenied` — Insufficient permissions
- `ResourceAlreadyExists` — Duplicate creation attempt

### Direct REST API Access

For operations not yet in the SDK, use `w.api_client.do()` with the HTTP method and API path. Prefer SDK methods when available.

## Code Examples

### Authentication

```python
"""
Databricks SDK Authentication Examples

Documentation: https://databricks-sdk-py.readthedocs.io/en/latest/authentication.html
"""

from databricks.sdk import WorkspaceClient, AccountClient

# Pattern 1: Environment Variables (Recommended)
# Set DATABRICKS_HOST and DATABRICKS_TOKEN, then:
w = WorkspaceClient()

# Verify connection
me = w.current_user.me()
print(f"Authenticated as: {me.user_name}")


# Pattern 2: Explicit Token Authentication
w = WorkspaceClient(
    host="https://your-workspace.cloud.databricks.com",
    token="dapi..."
)


# Pattern 3: Named Profile from ~/.databrickscfg
w = WorkspaceClient(profile="MY_PROFILE")


# Pattern 4: Azure Service Principal
w = WorkspaceClient(
    host="https://adb-123456789.azuredatabricks.net",
    azure_workspace_resource_id="/subscriptions/xxx/resourceGroups/xxx/...",
    azure_tenant_id="your-tenant-id",
    azure_client_id="your-client-id",
    azure_client_secret="your-client-secret"
)


# Pattern 5: Account-Level Client (users, workspaces, billing)
a = AccountClient(
    host="https://accounts.cloud.databricks.com",
    account_id="your-account-id",
    token="dapi..."
)

for workspace in a.workspaces.list():
    print(f"Workspace: {workspace.workspace_name}")


# Pattern 6: Auto-detected inside Databricks Notebooks
# from databricks.sdk import WorkspaceClient
# w = WorkspaceClient()  # Works automatically with notebook context
```

### Clusters and Jobs

```python
"""
Databricks SDK - Clusters and Jobs Examples

Clusters API: https://databricks-sdk-py.readthedocs.io/en/latest/workspace/compute/clusters.html
Jobs API: https://databricks-sdk-py.readthedocs.io/en/latest/workspace/jobs/jobs.html
"""

from datetime import timedelta
from databricks.sdk import WorkspaceClient
from databricks.sdk.service.compute import AutoScale
from databricks.sdk.service.jobs import Task, NotebookTask, JobCluster, ClusterSpec

w = WorkspaceClient()

# --- Clusters ---

# List all clusters
for cluster in w.clusters.list():
    print(f"{cluster.cluster_name}: {cluster.state}")

# Select best Spark version and node type automatically
spark_version = w.clusters.select_spark_version(latest=True, long_term_support=True)
node_type = w.clusters.select_node_type(local_disk=True, min_memory_gb=16)

# Create cluster with autoscaling (blocking)
cluster = w.clusters.create_and_wait(
    cluster_name="autoscale-cluster",
    spark_version=spark_version,
    node_type_id=node_type,
    autoscale=AutoScale(min_workers=1, max_workers=4),
    timeout=timedelta(minutes=30)
)

# Start / stop / delete
w.clusters.start(cluster_id="...").result()
w.clusters.stop(cluster_id="...")
w.clusters.delete(cluster_id="...")
w.clusters.ensure_cluster_is_running(cluster_id="...")


# --- Jobs ---

# List all jobs
for job in w.jobs.list():
    print(f"{job.job_id}: {job.settings.name}")

# Create a job with notebook task
created = w.jobs.create(
    name="my-notebook-job",
    tasks=[
        Task(
            task_key="main",
            notebook_task=NotebookTask(
                notebook_path="/Users/me/my-notebook",
                base_parameters={"param1": "value1"}
            ),
            existing_cluster_id="0123-456789-abcdef"
        )
    ]
)

# Run job and wait for completion
run = w.jobs.run_now_and_wait(
    job_id=created.job_id,
    notebook_params={"date": "2024-01-01"},
    timeout=timedelta(hours=1)
)
print(f"Run {run.run_id} finished: {run.state.result_state}")

# Submit one-time run (without creating a persistent job)
run = w.jobs.submit_and_wait(
    run_name="one-time-run",
    tasks=[
        Task(
            task_key="main",
            existing_cluster_id="...",
            notebook_task=NotebookTask(notebook_path="/Users/me/notebook")
        )
    ],
    timeout=timedelta(hours=1)
)
```

### SQL and Warehouses

```python
"""
Databricks SDK - SQL Warehouses and Statement Execution Examples

Warehouses API: https://databricks-sdk-py.readthedocs.io/en/latest/workspace/sql/warehouses.html
Statement Execution: https://databricks-sdk-py.readthedocs.io/en/latest/workspace/sql/statement_execution.html
"""

from datetime import timedelta
from databricks.sdk import WorkspaceClient
from databricks.sdk.service.sql import (
    StatementState,
    Disposition,
    Format,
    StatementParameterListItem,
)

w = WorkspaceClient()

# --- Warehouses ---

# List all warehouses
for warehouse in w.warehouses.list():
    print(f"{warehouse.name}: {warehouse.state} (id: {warehouse.id})")

# Create a serverless SQL warehouse
created = w.warehouses.create_and_wait(
    name="my-warehouse",
    cluster_size="Small",
    max_num_clusters=1,
    auto_stop_mins=15,
    enable_serverless_compute=True,
    timeout=timedelta(minutes=20)
)

# Start / stop
w.warehouses.start(id="abc123").result()
w.warehouses.stop(id="abc123").result()


# --- Statement Execution ---

# Execute a simple SQL query
response = w.statement_execution.execute_statement(
    warehouse_id="abc123",
    statement="SELECT * FROM main.default.my_table LIMIT 10",
    wait_timeout="30s"
)

if response.status.state == StatementState.SUCCEEDED:
    columns = [col.name for col in response.manifest.schema.columns]
    print(f"Columns: {columns}")
    for row in response.result.data_array:
        print(row)

# Parameterized query (prevents SQL injection)
response = w.statement_execution.execute_statement(
    warehouse_id="abc123",
    statement="SELECT * FROM users WHERE age > :min_age AND name = :name",
    parameters=[
        StatementParameterListItem(name="min_age", value="21", type="INT"),
        StatementParameterListItem(name="name", value="Alice", type="STRING"),
    ],
    wait_timeout="30s"
)

# Large results with external links
response = w.statement_execution.execute_statement(
    warehouse_id="abc123",
    statement="SELECT * FROM large_table",
    disposition=Disposition.EXTERNAL_LINKS,
    format=Format.ARROW_STREAM,
    wait_timeout="0s"
)
```

### Unity Catalog

```python
"""
Databricks SDK - Unity Catalog Examples

Catalogs: https://databricks-sdk-py.readthedocs.io/en/latest/workspace/catalog/catalogs.html
Schemas: https://databricks-sdk-py.readthedocs.io/en/latest/workspace/catalog/schemas.html
Tables: https://databricks-sdk-py.readthedocs.io/en/latest/workspace/catalog/tables.html
Volumes: https://databricks-sdk-py.readthedocs.io/en/latest/workspace/catalog/volumes.html
"""

from databricks.sdk import WorkspaceClient
from databricks.sdk.service.catalog import VolumeType

w = WorkspaceClient()

# --- Catalogs ---

for catalog in w.catalogs.list():
    print(f"Catalog: {catalog.name} (owner: {catalog.owner})")

new_catalog = w.catalogs.create(name="my_catalog", comment="My analytics catalog")


# --- Schemas ---

for schema in w.schemas.list(catalog_name="main"):
    print(f"Schema: {schema.full_name}")

new_schema = w.schemas.create(
    name="analytics", catalog_name="main", comment="Analytics data schema"
)


# --- Tables ---

for table in w.tables.list(catalog_name="main", schema_name="default"):
    print(f"Table: {table.full_name} ({table.table_type})")

table = w.tables.get(full_name="main.default.my_table")
for col in table.columns:
    print(f"  {col.name}: {col.type_name} (nullable: {col.nullable})")

exists = w.tables.exists(full_name="main.default.my_table")


# --- Volumes ---

for volume in w.volumes.list(catalog_name="main", schema_name="default"):
    print(f"Volume: {volume.full_name} ({volume.volume_type})")

managed_volume = w.volumes.create(
    catalog_name="main",
    schema_name="default",
    name="my_managed_volume",
    volume_type=VolumeType.MANAGED,
    comment="Managed volume for data files"
)

# Upload / download files to volumes
w.files.upload(
    file_path="/Volumes/main/default/my_volume/data.csv",
    contents=open("local_file.csv", "rb"),
    overwrite=True
)

for entry in w.files.list_directory_contents("/Volumes/main/default/my_volume/"):
    print(f"{entry.name}: {'dir' if entry.is_directory else 'file'}")
```

### Serving Endpoints and Vector Search

```python
"""
Databricks SDK - Model Serving and Vector Search Examples

Serving Endpoints: https://databricks-sdk-py.readthedocs.io/en/latest/workspace/serving/serving_endpoints.html
Vector Search: https://databricks-sdk-py.readthedocs.io/en/latest/workspace/vectorsearch/vector_search_indexes.html
"""

import json
from datetime import timedelta
from databricks.sdk import WorkspaceClient
from databricks.sdk.service.serving import (
    EndpointCoreConfigInput,
    ServedEntityInput,
    TrafficConfig,
    Route,
)

w = WorkspaceClient()

# --- Model Serving ---

# List endpoints
for endpoint in w.serving_endpoints.list():
    print(f"{endpoint.name}: {endpoint.state}")

# Create a serving endpoint for a Unity Catalog model
created = w.serving_endpoints.create_and_wait(
    name="my-model-endpoint",
    config=EndpointCoreConfigInput(
        served_entities=[
            ServedEntityInput(
                entity_name="main.ml.my_model",
                entity_version="1",
                workload_size="Small",
                scale_to_zero_enabled=True,
            )
        ],
        traffic_config=TrafficConfig(
            routes=[Route(served_model_name="my_model-1", traffic_percentage=100)]
        ),
    ),
    timeout=timedelta(minutes=30)
)

# Query custom model endpoint
response = w.serving_endpoints.query(
    name="my-model-endpoint",
    inputs=[{"feature1": 1.0, "feature2": "value"}]
)

# Query chat/completions endpoint (LLM)
response = w.serving_endpoints.query(
    name="my-llm-endpoint",
    messages=[
        {"role": "system", "content": "You are a helpful assistant."},
        {"role": "user", "content": "Hello!"}
    ],
    max_tokens=100
)

# Get OpenAI-compatible client
openai_client = w.serving_endpoints.get_open_ai_client()


# --- Vector Search ---

# List indexes
for index in w.vector_search_indexes.list_indexes(endpoint_name="my-vs-endpoint"):
    print(f"Index: {index.name}")

# Query with text
results = w.vector_search_indexes.query_index(
    index_name="main.default.my_index",
    columns=["id", "text", "metadata"],
    query_text="What is machine learning?",
    num_results=5,
    filters_json='{"category": "ai"}'
)

for doc in results.result.data_array:
    print(f"Score: {doc[-1]}, Text: {doc[1][:100]}...")

# Upsert data into a Direct Vector Access index
w.vector_search_indexes.upsert_data_vector_index(
    index_name="main.default.direct_index",
    inputs_json=json.dumps([
        {"id": "1", "text": "Hello world", "embedding": [0.1, 0.2, 0.3]},
        {"id": "2", "text": "Another doc", "embedding": [0.4, 0.5, 0.6]},
    ])
)

# Sync a Delta Sync index
w.vector_search_indexes.sync_index(index_name="main.default.delta_sync_index")
```
