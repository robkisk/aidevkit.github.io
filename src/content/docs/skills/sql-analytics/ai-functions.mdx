## Overview

The AI Functions skill teaches Claude Code how to use Databricks built-in AI functions that call Foundation Model APIs directly from SQL and PySpark pipelines. These functions work on table columns as naturally as `UPPER()` or `LENGTH()`, eliminating the need to set up model endpoints, manage API keys, or write inference boilerplate. Use it when you need to enrich data with classification, extraction, summarization, translation, or forecasting at scale.

## Capabilities

- Classify text into fixed categories with `ai_classify` (2-20 labels)
- Extract structured entities from unstructured text with `ai_extract`
- Summarize text with controllable length using `ai_summarize`
- Detect sentiment with `ai_analyze_sentiment`
- Redact PII (names, emails, phone numbers, addresses) with `ai_mask`
- Translate text across supported languages with `ai_translate`
- Fix grammar and spelling with `ai_fix_grammar`
- Generate free-form text with `ai_gen`
- Measure semantic similarity between text pairs with `ai_similarity`
- Parse PDFs and Office documents with `ai_parse_document`
- Run complex structured extraction with `ai_query` and custom model endpoints
- Forecast time series data with `ai_forecast`
- Build end-to-end document processing pipelines (parse, chunk, index, query)

## Key Concepts

### Task-Specific vs General-Purpose Functions

Databricks provides two tiers of AI functions:

**Task-specific functions** â€” Optimized for common NLP tasks. Always prefer these when they fit your use case:
- `ai_analyze_sentiment`, `ai_classify`, `ai_extract`, `ai_fix_grammar`
- `ai_gen`, `ai_mask`, `ai_similarity`, `ai_summarize`, `ai_translate`
- `ai_parse_document`

**General-purpose function** â€” `ai_query` sends arbitrary prompts to any model endpoint. Use it only when no task-specific function applies, such as complex nested JSON extraction or multi-step reasoning.

**Table-valued function** â€” `ai_forecast` generates time series predictions and returns a result table with forecast values and confidence intervals.

### Function Selection Rule

Always prefer a task-specific function over `ai_query`. For example, use `ai_classify` for categorization (not `ai_query` with a classification prompt), and `ai_summarize` for summarization (not `ai_query` with a summarize instruction).

### Prerequisites

- Databricks SQL warehouse (not Classic) or cluster with DBR 15.1+
- DBR 15.4 ML LTS recommended for batch workloads
- DBR 17.1+ required for `ai_parse_document`
- `ai_forecast` requires a Pro or Serverless SQL warehouse
