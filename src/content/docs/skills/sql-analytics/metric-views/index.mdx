## Overview

The Metric Views skill teaches Claude Code how to define reusable, governed business metrics using YAML in Unity Catalog. Metric views separate measure definitions from dimension groupings, enabling flexible querying at runtime. Use them when building standardized KPIs, revenue metrics, or any reusable business metrics that need consistent definitions across teams and tools.

## Capabilities

- Define standardized metrics with dimensions and aggregate measures
- Create complex aggregations: ratios, distinct counts, filtered measures
- Build star and snowflake schema joins declaratively in YAML
- Define window measures for moving averages, running totals, and period-over-period comparisons
- Enable materialization for pre-computed metric aggregations
- Integrate natively with AI/BI Dashboards, Genie, and Alerts

## Key Concepts

### Dimensions vs Measures

| | Dimensions | Measures |
|---|---|---|
| Purpose | Categorize and group | Aggregate numeric values |
| Examples | Region, Date, Status | `SUM(revenue)`, `COUNT(orders)` |
| In queries | SELECT and GROUP BY | Wrapped in `MEASURE()` |
| Expressions | Any SQL expression | Must use aggregate functions |

### Quick Start

```sql
CREATE OR REPLACE VIEW catalog.schema.orders_metrics
WITH METRICS LANGUAGE YAML
AS $$
  version: 1.1
  source: catalog.schema.orders
  dimensions:
    - name: Order Month
      expr: DATE_TRUNC('MONTH', order_date)
    - name: Order Status
      expr: status
  measures:
    - name: Total Revenue
      expr: SUM(total_price)
    - name: Order Count
      expr: COUNT(1)
$$
```

### Query Pattern

All measures must use the `MEASURE()` function. `SELECT *` is NOT supported.

```sql
SELECT `Order Month`, `Order Status`,
  MEASURE(`Total Revenue`) AS total_revenue,
  MEASURE(`Order Count`) AS order_count
FROM catalog.schema.orders_metrics
GROUP BY ALL ORDER BY ALL
```

### Prerequisites

- Databricks Runtime 17.2+ (for YAML version 1.1)
- SQL warehouse with `CAN USE` permissions
- `SELECT` on source tables, `CREATE TABLE` + `USE SCHEMA` in target schema
