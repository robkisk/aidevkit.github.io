## Overview

The Databricks SQL skill teaches Claude Code how to use advanced DBSQL features including SQL scripting with procedural logic, stored procedures, recursive CTEs, materialized views with scheduled refresh, pipe syntax for readable query chains, geospatial analysis with H3 and ST functions, collations for case-insensitive search, and AI functions for LLM-powered data enrichment. Use it when building complex SQL pipelines, dimensional models, or leveraging any advanced SQL capability on Databricks.

## Capabilities

- Write SQL scripts with `BEGIN...END` blocks, variables, control flow, and exception handling
- Create and call stored procedures with `IN`/`OUT`/`INOUT` parameters
- Build recursive CTEs for hierarchy traversal, graph walking, and series generation
- Create materialized views with scheduled, event-driven, or manual refresh
- Chain SQL operations using pipe syntax (`|>`) for readable transformations
- Perform geospatial analysis with 39 H3 functions and 80+ ST spatial functions
- Configure collations for case-insensitive and accent-insensitive string matching
- Enrich data with AI functions (`ai_classify`, `ai_extract`, `ai_query`, etc.)
- Call external APIs from SQL with `http_request` and `remote_query`
- Ingest raw files with `read_files` table-valued function
- Apply data modeling best practices with star schemas and Liquid Clustering

### Data Modeling Approach

Use a hybrid medallion architecture:
- **Silver layer**: OBT or Data Vault for rapid integration and cleansing
- **Gold layer**: Star schema dimensional models as the curated, business-ready presentation layer for BI and reporting

**Liquid Clustering** is the default recommendation for all new Delta tables, replacing both partitioning and Z-ORDER. Choose 1-4 clustering keys based on the most frequently filtered columns.

### Key Guidelines

- Always use Serverless SQL warehouses for AI functions, MVs, and `http_request`
- Prefer Liquid Clustering over partitioning for new tables (1-4 keys max)
- Use `CLUSTER BY AUTO` when unsure about clustering keys
- Star schema in Gold layer for BI; OBT acceptable in Silver
- Define PK/FK constraints on dimensional models for query optimization
- Use `COLLATE UTF8_LCASE` for user-facing string columns that need case-insensitive search
