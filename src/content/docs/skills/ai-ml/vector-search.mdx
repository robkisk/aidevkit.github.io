## Overview

The Vector Search skill teaches Claude Code how to create and manage Databricks Vector Search endpoints and indexes for RAG (Retrieval-Augmented Generation) applications and semantic search. Use it when building knowledge bases, implementing similarity matching, or adding retrieval capabilities to AI agents. It covers endpoint selection, index creation, query patterns, and integration with the broader Databricks AI stack.

## Capabilities

- Create Standard endpoints (low-latency, ~50-100ms) and Storage-Optimized endpoints (cost-effective, 1B+ vectors)
- Build Delta Sync indexes with managed embeddings (Databricks computes vectors automatically)
- Build Delta Sync indexes with self-managed embeddings (bring your own vectors)
- Create Direct Access indexes for real-time CRUD operations
- Query indexes with text, vectors, or hybrid search (semantic + keyword)
- Apply filters using dictionary syntax (Standard) or SQL-like syntax (Storage-Optimized)
- Trigger index syncs for on-demand refresh
- Integrate with Databricks agents via `VectorSearchRetrieverTool`

## Key Concepts

### Endpoint Types

| Type | Latency | Capacity | Best For |
|------|---------|----------|----------|
| **Standard** | ~50-100ms | 320M vectors (768 dim) | Real-time applications needing low latency |
| **Storage-Optimized** | ~250ms | 1B+ vectors (768 dim) | Large-scale search, cost-sensitive workloads |

Storage-Optimized endpoints are newer and recommended for most use cases unless sub-100ms latency is required.

### Index Types

**Delta Sync with managed embeddings** — Provide a text column and an embedding model; Databricks computes and maintains the vectors automatically. Easiest setup.

**Delta Sync with self-managed embeddings** — Provide a pre-computed embedding vector column. Use when you need custom embedding models or have existing vectors.

**Direct Access** — Manual insert/update/delete operations on vectors. Use for real-time applications that need immediate updates without waiting for a sync cycle.

### Hybrid Search

Combines vector similarity with keyword matching for better recall. Available on both Delta Sync and Direct Access indexes. Enable it by passing `query_type="hybrid"` in your query.

### Embedding Models

Databricks provides built-in embedding models for managed indexes:

| Model | Dimensions | Best For |
|-------|------------|----------|
| `databricks-gte-large-en` | 1024 | English text, high quality |
| `databricks-bge-large-en` | 1024 | English text, general purpose |

### Filter Syntax

Filter syntax varies by endpoint type:
- **Standard endpoints** use `filters_json` with dictionary format: `{"category": "ai"}`
- **Storage-Optimized endpoints** use `filter_string` with SQL-like syntax: `"category = 'ai' AND year > 2024"`
