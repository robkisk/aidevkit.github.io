## Overview

The Agent Bricks skill teaches Claude Code how to create pre-built AI components on Databricks. Agent Bricks include Knowledge Assistants for RAG-based document Q&A, Genie Spaces for natural language SQL, and Supervisor Agents that orchestrate multiple specialized agents. Use it when building conversational AI applications that combine document retrieval, data analytics, ML classification, and external tool integration.

## Capabilities

- Create Knowledge Assistants (KA) from documents in Unity Catalog Volumes
- Set up Genie Spaces for SQL-based data exploration
- Build Supervisor Agents (MAS) that route queries across multiple specialized agents
- Integrate UC Functions and external MCP servers as agent tools
- Auto-extract examples from JSON files in document volumes
- Monitor provisioning status and endpoint readiness

## Related MCP Tools

| Tool | Description |
|------|-------------|
| `manage_ka` | Create, get, find, or delete Knowledge Assistants |
| `manage_mas` | Create, get, find, or delete Supervisor Agents |

| `get_genie` | Get Genie Space details |

## Key Concepts

### Agent Brick Types

| Brick | Purpose | Data Source |
|-------|---------|-------------|
| Knowledge Assistant (KA) | Document-based Q&A using RAG | PDF/text files in Volumes |
| Genie Space | Natural language to SQL | Unity Catalog tables |
| Supervisor Agent (MAS) | Multi-agent orchestration | Model serving endpoints |

### Agent Types in Supervisor

Each agent in a Supervisor must use exactly one of:
- `ka_tile_id` -- Knowledge Assistant
- `genie_space_id` -- Genie Space
- `endpoint_name` -- Custom model serving endpoint
- `uc_function_name` -- Unity Catalog function
- `connection_name` -- External MCP server

### Provisioning

Newly created KA and MAS tiles progress through: `PROVISIONING` (2-5 min) -> `ONLINE` (ready) -> `OFFLINE` (stopped).
