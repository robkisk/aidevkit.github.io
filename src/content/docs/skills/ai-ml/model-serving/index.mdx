## Overview

The Model Serving skill teaches Claude Code how to deploy MLflow models and AI agents to Databricks Model Serving endpoints. It covers classical ML models (sklearn, xgboost), custom Python models (pyfunc), and GenAI agents (ResponsesAgent, LangGraph). Use it when deploying models to REST endpoints, integrating Unity Catalog functions as agent tools, or querying deployed endpoints.

## Capabilities

- Deploy traditional ML models with `mlflow.sklearn.autolog()`
- Build custom Python models with `mlflow.pyfunc.PythonModel`
- Create GenAI agents using `ResponsesAgent` pattern
- Integrate UC Functions and Vector Search as agent tools
- Test agents locally before deployment
- Deploy endpoints asynchronously via Databricks Jobs
- Query chat, agent, and ML endpoints via SDK or MCP tools

## Key Concepts

### What Are You Deploying?

| Model Type | Pattern | Key File |
|------------|---------|----------|
| Traditional ML (sklearn, xgboost) | `mlflow.sklearn.autolog()` | Classical ML |
| Custom Python model | `mlflow.pyfunc.PythonModel` | Custom PyFunc |
| GenAI Agent (LangGraph, tool-calling) | `ResponsesAgent` | GenAI Agents |

### Foundation Model Endpoints

Use exact endpoint names. Common defaults:
- **Agent LLM**: `databricks-meta-llama-3-3-70b-instruct`
- **Embedding**: `databricks-gte-large-en`
- **Code tasks**: `databricks-gpt-5-1-codex-mini`

### ResponsesAgent Output Format

Use helper methods, not raw dicts:

```python
# CORRECT
return ResponsesAgentResponse(
    output=[self.create_text_output_item(text="...", id="msg_1")]
)
```

## Common Issues

| Issue | Solution |
|-------|----------|
| Invalid output format | Use `self.create_text_output_item()` not raw dicts |
| Endpoint NOT_READY | Deployment takes ~15 min; poll with `get_serving_endpoint_status` |
| Package not found | Specify exact versions in `pip_requirements` |
| Tool timeout | Use job-based deployment, not synchronous calls |
