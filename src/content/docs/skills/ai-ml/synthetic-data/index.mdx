**Skill:** `databricks-synthetic-data-gen`

## Overview

The Synthetic Data Generation skill teaches Claude Code how to generate realistic, story-driven synthetic data on Databricks using Spark + Faker + Pandas UDFs. It supports serverless execution, multiple output formats, and scales from thousands to millions of rows. Use it when creating demo datasets, test data, or populating tables for Genie Spaces and data pipelines.

## Capabilities

- Generate data with Spark + Faker + Pandas UDFs (scalable, parallel)
- Create realistic distributions: log-normal pricing, weighted tiers, seasonal time patterns
- Maintain referential integrity across related tables
- Support multiple output formats: Parquet, JSON, CSV, Delta tables
- Run on serverless compute with managed dependencies
- Add table and column comments for Unity Catalog discoverability

## Key Concepts

### Planning Workflow

1. **Gather requirements**: catalog/schema, domain, row counts, relationships
2. **Present table specification**: columns, descriptions, distributions, assumptions
3. **Confirm with user**: catalog, distributions, compute preference
4. **Generate**: master tables first, then related tables with valid FKs

### Critical Rules

- Always use Spark + Faker + Pandas UDFs (not pure Python)
- Never use `.cache()` or `.persist()` with serverless compute
- Always ask for catalog/schema -- never default
- Create infrastructure in script: `CREATE SCHEMA/VOLUME IF NOT EXISTS`
- Present a plan for user approval before generating code

### Common Patterns

```python
# Weighted tier distribution
F.when(F.rand() < 0.6, "Free")
 .when(F.rand() < 0.9, "Pro")
 .otherwise("Enterprise").alias("tier")

# Log-normal amounts (realistic pricing)
@F.pandas_udf(DoubleType())
def generate_amount(tiers: pd.Series) -> pd.Series:
    return pd.Series([
        float(np.random.lognormal({"Enterprise": 7.5, "Pro": 5.5, "Free": 4.0}[t], 0.7))
        for t in tiers
    ])
```
