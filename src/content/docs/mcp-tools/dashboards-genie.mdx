These tools manage Databricks AI/BI dashboards and Genie spaces — creating visualizations, publishing dashboards, asking natural language questions, and defining reusable metric views.

## create_or_update_dashboard

**Description:** Creates a new AI/BI dashboard or updates an existing one. Handles the full dashboard definition including datasets, visualizations, layout, and filters.

**Parameters:**

| Parameter | Type | Required | Description |
|---|---|---|---|
| display_name | string | Yes | Dashboard display name |
| warehouse_id | string | No | SQL warehouse for dashboard queries |
| serialized_dashboard | object | No | Full dashboard definition (datasets, pages, widgets) |
| parent_path | string | No | Workspace folder path for the dashboard |
| dashboard_id | string | No | Existing dashboard ID to update (omit for create) |

**Example prompt:**
```
Create a dashboard called "Sales Overview" with charts showing
monthly revenue, top products, and customer segmentation
```

**Related skills:** [AI/BI Dashboards](/aidevkit.github.io/skills/sql-analytics/aibi-dashboards/)

## publish_dashboard

**Description:** Publishes a draft dashboard, making it available to viewers. Optionally sets the warehouse used for published queries and enables/disables embedded credentials.

**Parameters:**

| Parameter | Type | Required | Description |
|---|---|---|---|
| dashboard_id | string | Yes | The dashboard to publish |
| warehouse_id | string | No | SQL warehouse for published queries |
| embed_credentials | boolean | No | Whether to embed credentials so viewers can run queries without their own warehouse access |

**Example prompt:**
```
Publish the Sales Overview dashboard with embedded credentials
so stakeholders can view it without warehouse access
```

**Related skills:** [AI/BI Dashboards](/aidevkit.github.io/skills/sql-analytics/aibi-dashboards/)

## create_or_update_genie

**Description:** Creates a new Genie space or updates an existing one. Genie spaces allow users to ask natural language questions about their data, powered by AI that understands the underlying tables and relationships.

**Parameters:**

| Parameter | Type | Required | Description |
|---|---|---|---|
| display_name | string | Yes | Genie space display name |
| warehouse_id | string | No | SQL warehouse for query execution |
| table_identifiers | string[] | No | Unity Catalog table names to make queryable |
| description | string | No | Description of the data and its intended use |
| instructions | string[] | No | Custom instructions for the AI (business rules, terminology) |
| genie_space_id | string | No | Existing space ID to update (omit for create) |

**Example prompt:**
```
Create a Genie space for the sales team that covers the orders,
customers, and products tables with instructions about our
fiscal year starting in February
```

**Related skills:** [Genie Spaces](/aidevkit.github.io/skills/sql-analytics/genie-spaces/)

**Example prompt:**

## manage_metric_views

**Description:** Multi-action tool for managing metric views — reusable metric definitions that provide consistent calculations across dashboards, Genie spaces, and SQL queries.

**Parameters:**

| Parameter | Type | Required | Description |
|---|---|---|---|
| action | string | Yes | The operation: `create`, `get`, `list`, `update`, or `delete` |
| name | string | No | Metric view name (three-level: `catalog.schema.name`) |
| definition | object | No | Metric view YAML definition including measures, dimensions, and filters |

**Example prompt:**
```
Create a metric view for monthly recurring revenue (MRR)
based on the subscriptions table with dimensions for
region and product tier
```

**Related skills:** [Metric Views](/aidevkit.github.io/skills/sql-analytics/metric-views/)
