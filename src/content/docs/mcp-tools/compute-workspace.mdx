These tools manage Databricks compute resources (clusters), Unity Catalog volumes (file storage), workspace files, and resource tracking. They handle the infrastructure layer that other tools depend on.

## list_clusters

**Description:** Lists all clusters in the workspace with their current state, configuration, creator, and runtime version.

**Parameters:**

| Parameter | Type | Required | Description |
|---|---|---|---|
| — | — | — | No parameters required |

**Example prompt:**
```
Show me all the clusters in my workspace and their status
```

**Related skills:** [Workspace Config](/aidevkit.github.io/skills/devops-config/workspace-config/)

## manage_workspace

**Description:** Multi-action tool for managing workspace objects — notebooks, folders, files, and repos. Supports listing, importing, exporting, creating, and deleting workspace items.

**Parameters:**

| Parameter | Type | Required | Description |
|---|---|---|---|
| action | string | Yes | The operation: `list`, `get_status`, `import`, `export`, `delete`, `mkdirs` |
| path | string | Yes | Workspace path (e.g., `/Workspace/Users/me/notebook`) |
| content | string | No | Base64-encoded content for import |
| format | string | No | Import/export format: `SOURCE`, `HTML`, `JUPYTER`, `DBC` |
| language | string | No | Language for imports: `PYTHON`, `SQL`, `SCALA`, `R` |
| overwrite | boolean | No | Whether to overwrite existing objects |

**Example prompt:**
```
List all notebooks in my workspace home folder
```

**Related skills:** [Workspace Config](/aidevkit.github.io/skills/devops-config/workspace-config/)

## upload_file

**Description:** Uploads a local file to the Databricks workspace. The file is placed at the specified workspace path.

**Parameters:**

| Parameter | Type | Required | Description |
|---|---|---|---|
| local_path | string | Yes | Path to the local file |
| remote_path | string | Yes | Destination workspace path |
| overwrite | boolean | No | Whether to overwrite an existing file |

**Example prompt:**
```
Upload my local pipeline.py to /Workspace/Shared/pipelines/
```

## upload_to_volume

**Description:** Uploads a local file to a Unity Catalog volume. Volumes provide managed cloud storage integrated with UC governance.

**Parameters:**

| Parameter | Type | Required | Description |
|---|---|---|---|
| local_path | string | Yes | Path to the local file |
| volume_path | string | Yes | Destination volume path (e.g., `/Volumes/catalog/schema/volume/file.csv`) |
| overwrite | boolean | No | Whether to overwrite an existing file |

**Example prompt:**
```
Upload the sample_data.csv file to the raw-data volume
in the analytics catalog
```

**Related skills:** [Unity Catalog - Volumes](/aidevkit.github.io/skills/governance-catalog/unity-catalog/volumes/)
