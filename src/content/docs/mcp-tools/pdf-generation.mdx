Generate synthetic PDF documents and upload them to Unity Catalog volumes for RAG testing and evaluation.

### Example prompts

> "Generate 10 HR policy PDFs and upload them to my raw_data volume"

> "Create a single API authentication guide PDF with a question and guideline for RAG evaluation"

---

## generate_and_upload_pdf

**Description:** Convert HTML to PDF and upload to a Unity Catalog volume.

**Parameters:**

| Parameter | Type | Required | Description |
|---|---|---|---|
| `html_content` | `str` | Yes | Complete HTML document including &lt;!DOCTYPE html&gt;, &lt;html&gt;, &lt;head&gt;, &lt;style&gt;, and &lt;body&gt; tags. Use modern CSS3 for styling. |
| `filename` | `str` | Yes | Name for the PDF file (e.g., "report.pdf" or "report" - .pdf added if missing) |
| `catalog` | `str` | Yes | Unity Catalog name |
| `schema` | `str` | Yes | Schema name |
| `volume` | `str` | No | Volume name (must already exist). Default: "raw_data" |
| `folder` | `Optional[str]` | No | Optional folder within volume (e.g., "documents") |
